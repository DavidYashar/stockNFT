# Project Rules ‚Äî Google Stock NFT

## Core Principles
- **One source of truth** for everything: prices (Yahoo Finance `/api/googl-price`), addresses (`.env`), data (on-chain state)
- **Never assume sequential token IDs** ‚Äî always use `tokenByIndex()` (ERC-721Enumerable). Token IDs can have gaps due to `burnUnminted`.
- **Verify auth modifier before calling**: if a function is `onlyOwner`, check who the owner is first
- **Deployer (0x2bAF...) = mint controls only**. Treasury vault (0xF772...) = DeFi operations (sweep, harvest, Google purchase, interest distribution)
- **NO ICONS/EMOJIS in UI text** ‚Äî no icons next to titles, labels, buttons, or anywhere in the interface. Clean text only.

## Contracts
- 5 core: GoogleStockNFT, PlatformManager, StockVault, InterestDistributor, Marketplace
- 3 mocks: MockGOOGLon, MockUSDC, MockAavePool
- PM.recordSweep/recordHarvest/triggerGooglePurchase: `onlySweepOrOwner` modifier (owner or sweepOperator)

### Loyalty Fee Bot (`loyalty-bot.ts`) ‚Äî ‚úÖ Verified Working
**Why the bot exists**: When a buyer purchases an NFT on the Marketplace, EIP-2981 royalty (10%) is sent as an **internal ETH transfer** from marketplace contract ‚Üí treasury EOA. The marketplace contract CANNOT call `PM.receiveLoyalty()` directly because PM's auth (`onlyOwner`/`onlySweepOrOwner`) rejects the marketplace address. So the bot watches for these internal transfers and records them on-chain.

**How it works**:
1. Polls Blockscout `txlistinternal` API (`module=account&action=txlistinternal`) for the treasury address
2. Filters for internal transfers where `from` = marketplace contract AND `to` = treasury EOA
3. Deduplicates using composite key: `blockNumber + from + to + value` ‚Äî prevents double-counting
4. Calls `PM.receiveLoyalty()` via the **deployer private key** (deployer is PM owner and can call it)
5. Stores record in `backend/data/loyalty-fees.json` with `{amount, txHash, recorded: true|false}`

**Key details**:
- API: `https://eth-sepolia.blockscout.com/api` (Sepolia testnet)
- Bot runs on port 3002 (Express)
- Composite dedup key: `${blockNumber}-${from}-${to}-${value}` ‚Üí stored in a Set to skip already-seen transfers
- `recorded: true` means PM.receiveLoyalty() transaction succeeded on-chain
- `PM.totalLoyaltyFees` is the on-chain source of truth ‚Äî always check this, not the bot data file
- Verified 2026-06-18: 2 marketplace buys correctly detected (blocks 11086431, 11086433), both 0.001 ETH, PM.totalLoyaltyFees = 0.002 ETH ‚úÖ

### Interest Distributor Flow (`InterestDistributor.sol`) ‚Äî ‚úÖ Verified Working
**Purpose**: Receives DeFi yield from MockAavePool (via Treasury Vault) and distributes it equally among all NFT holders after mint ends.

**Flow**:
1. Treasury vault harvests yield from MockAavePool ‚Üí receives ETH
2. Treasury vault calls `ID.fundEqualDistribution()` with ETH (`onlyTreasuryVault` modifier)
3. ID reads `NFT.totalSupply()` on-chain, computes `perToken = msg.value / totalSupply`
4. After `allowClaims()` is called (by treasury vault as owner), NFT holders call `claimInterest(tokenId)`
5. Each holder receives `distributionPerToken` ETH per token they own, once per round

**Key contract details**:
- ID owner = treasury vault (0xF772...)
- `fundEqualDistribution()`: `onlyTreasuryVault` ‚Äî only treasury can fund
- `allowClaims()`: `onlyOwner` ‚Äî only treasury vault can enable claims
- `claimInterest(tokenId)`: verifies `ownerOf(tokenId) == msg.sender` via staticcall to NFT
- `lastClaimedRound[tokenId] < distributionRound`: prevents double-claiming per round
- `interestPool` tracks total undistributed ETH; reduced by `perToken` on each claim
- Uses `payable(msg.sender).transfer(amount)` ‚Äî appears as **internal transaction** on block explorers

**Math**: Equal distribution model ‚Äî `perToken = fundedAmount / totalSupply`. All holders get the same amount per NFT regardless of when they minted.

**On-chain source of truth**: `ID.interestPool`, `ID.distributionPerToken`, `ID.distributionRound`, `ID.lastClaimedRound(tokenId)` ‚Äî always check these, not external APIs.

**Verified 2026-06-18**: Treasury funded 0.001 ETH (block 11086511), 5 NFTs exist ‚Üí 0.0002 ETH per token. Deployer (owns all 5) claimed all 5 via `claimInterest()` at blocks 11086518-11086527. 5 internal transfers of 0.0002 ETH each confirmed. Interest pool drained to 0. Math: 0.001 √∑ 5 = 0.0002 ‚úÖ

### Google Purchase Flow (`StockVault.sol`) ‚Äî ‚öÝÔ∏è Requires Manual Funding Step
**Purpose**: One-time GOOGLon purchase using pool80 (80% of mint revenue) + loyalty fees. GOOGLon is then distributed proportionally to NFT holders for redemption.

**Prerequisites**: `mintEnded == true` AND `totalLoyaltyFees >= gap20()` (= 20% of totalMintPrincipal).

**Flow (3 steps, all from Treasury Vault wallet)**:
1. **Fund Pool80**: `SV.receivePool80Funds{value: pool80}()` ‚Äî sends 80% Google budget to StockVault
2. **Fund Loyalty**: `SV.receiveLoyaltyFunds{value: loyaltyFees}()` ‚Äî sends loyalty fees to StockVault
3. **Trigger**: `PM.triggerGooglePurchase()` ‚Üí calls `SV.executeGooglePurchase(totalMintPrincipal)` ‚Üí `_simulateSwap()` mints GOOGLon

**Simulation formula** (testnet): `GOOGLon = ETH √ó 2000 / 365` (assumes 1 ETH ‚âà $2000, GOOGL ‚âà $365)

**Share distribution** (after trigger): `PM.assignShares(tokenIds, principals, totalPrincipal)` ‚Üí each NFT gets `totalGOOGLon √ó mintPrincipal_i / totalMintPrincipal`

**Redemption**: Holder calls `SV.requestRedemption(tokenId)` ‚Üí burns NFT ‚Üí waits `redemptionDelay` (48s testnet) ‚Üí `SV.claimRedemption(tokenId)` ‚Üí receives GOOGLon minus 5% fee

**Key auth**:
- `SV.receivePool80Funds()`: owner, platformManager, or treasuryVault
- `SV.receiveLoyaltyFunds()`: owner or platformManager
- `SV.executeGooglePurchase()`: `onlyPlatformManager` (called by PM)
- `PM.triggerGooglePurchase()`: `onlySweepOrOwner` (treasury vault or deployer)

**Critical**: `SV.executeGooglePurchase()` requires `address(this).balance >= totalETH`. If SV has 0 ETH, the trigger reverts with "Swap failed". Always fund SV first (Steps 1 & 2) before triggering (Step 3).

**`usdcToken` placeholder**: SV's `usdcToken` is set to deployer address intentionally ‚Äî the ETH-native flow never uses it. Only matters for a future real Uniswap V4 swap.

**Admin page**: 3 buttons under "Google Purchase" ‚Äî Step 1 (Send Pool80), Step 2 (Send Loyalty), Step 3 (Trigger). SV balances displayed between steps. All Treasury Vault only.

**‚öÝÔ∏è feeRecipient must be set to Treasury**: SV constructor defaults `feeRecipient = _initialOwner` (deployer). Call `SV.setFeeRecipient(treasury)` once after deploy, otherwise 5% redemption fees go to deployer instead of treasury. One-time fix by treasury vault (SV owner).
